# 마스터 프롬프트 & 끊김 없는 롱폼 절차

## 단일 클립(≤15초) 영상 프롬프트 구조

순서대로: **[피사체] + [동작] + [배경] + [카메라] + [조명/분위기] + [스타일]**. 피사체·동작을 앞에, 조명·그레이드를 뒤에 둔다(모델이 앞부분에 더 집중).

구간별 카메라 동선을 명시한다. 예: `천천히 다가가기(0–5s) → 잠깐 멈춤(5–8s) → 천천히 빠지기(8–15s)`.

흔들림 방지·원테이크를 요청하고, 장면이 갑자기 튀지 않게 한다.

## 끊김 없는 롱폼(15~30초) 체이닝 절차 ★

목표: 시청자가 **하나의 연속된 영상**으로 느끼게 한다. 단순 이어붙이기 금지.

1. **하나의 모델로 통일** — 전 구간 동일 모델(기본 Seedance 2.0). 모델을 섞으면 룩이 튄다.
2. **레퍼런스 고정** — 같은 제품/인물 레퍼런스를 모든 클립에 동일하게 사용해 정체성을 유지.
3. **프레임 물리기(핵심)** — 1번 클립의 **마지막 프레임을 2번 클립의 start_image(시작 프레임)**로 사용한다(seedance/kling은 start_image·end_image role 지원). 이렇게 해야 이음매가 보이지 않는다.
4. **규격 통일** — 모든 클립의 비율·해상도·프레임 톤·색감(그레이드)을 동일하게.
5. **연속 카메라 동선** — 클립 경계에서 카메라가 끊기지 않게, 한 동작이 다음 클립으로 이어지는 동선을 설계(예: 클립1 끝에서 다가가던 카메라가 클립2 시작에서 계속 다가감).
6. **분할 설계** — 30초면 15초+15초(또는 3×10초). 각 구간에 역할 배정: 훅 → 베네핏 → CTA.
7. 각 클립 생성 전 get_cost로 비용 합계를 보여주고 승인받는다. 체이닝은 클립 수만큼 비용이 곱해지므로 특히 사전 고지가 중요.

> 모델이 end_image→start_image 체이닝을 지원하지 않거나 결과 이음매가 거슬리면, 클립 경계를 **컷 전환(같은 장소·같은 조명의 다른 앵글)**으로 자연스럽게 처리한다. 완전히 다른 두 장면을 억지로 붙이지 않는다.

## 사용자에게 주는 마스터 프롬프트(폼/대화 공용 골자)

```
- 제품 페이지/사진 : {product}
- 분위기 : {mood}
- 길이 : {length} (최대 30초, 길면 끊김 없이 체이닝)
- 비율 : {ratio}
- 광고 포맷 : {format}
- 영상 모델 : {video_model}   이미지 모델 : {image_model}
- 사진 : {photo_mode}   마무리 : {ending}
- 카피·자막 : 작성 (훅→베네핏→CTA, 금지표현 회피)

지침: S0 점검 → S1 결손정보 질문(추천 우선) → S2 분석 →
S3 이미지4컷 → S4 시작프레임+카피+자막+비용 승인 →
S5 끊김 없는 영상(≤15s 단일 / >15s 체이닝) → S6 전달.
모든 비용 생성 전 get_cost로 승인. 매 단계 추천안 먼저 제시.
```
