# 모드 B — 레퍼런스 영상 → 재현 프롬프트

사용자가 "이 영상처럼 만들어줘", "이 영상 분석해서 프롬프트 뽑아줘", 레퍼런스 영상 링크/파일을 줄 때 사용한다. **출력은 재현용 프롬프트까지**다. 영상 생성은 사용자가 승인하면 별도로(모드 A로) 진행한다.

## 먼저 고지할 것 (생성 전 반드시)

- **완벽 복제가 아니다.** 분석은 근사치이며, "이런 느낌으로 다시" 수준이다.
- **짧을수록 정확하다.** 영상이 길수록 장면 분석 정확도가 떨어진다. 30초 이내 클립을 권장한다.
- **저작권/상표 주의.** 경쟁사 광고·보호 저작물을 1:1로 복제하는 용도로는 쓰지 않는다. 스타일·구조 참고로 안내하고, 근접 복제 요청이면 위험을 알리고 변형을 제안한다.
- 분석은 3~5분 걸린다.

## 입력 처리

`video_analysis_create`는 **정확히 하나**만 받는다:
- **유튜브 링크** → `youtube_url`에 전달.
- **업로드 파일** → `media_upload_widget`으로 영상 업로드받고, 반환된 media_id를 `video_input_id`로 전달. (채팅 첨부·틱톡/인스타 링크는 직접 안 됨 → 파일을 내려받아 업로드 안내.)

그다음 `video_analysis_status`를 30~60초 간격으로 폴링해 `status='completed'`(scenes 채워짐)까지 기다린다.

## 분석 → 재현 프롬프트 변환

완료된 `scenes`(장면별 설명)를 다음 구조로 재구성한다. 장면마다:

`[피사체] · [동작] · [배경] · [카메라(앵글/무빙)] · [조명·컬러그레이드] · [스타일] · [타이밍(초)]`

그리고 영상 전체 수준에서 추정·명시:
- 화면 비율, 총 길이, 컷 수, 전환 방식(컷/디졸브/원테이크)
- 추천 모델(`references/models.md`): 인물·제품 일관성 필요 → Seedance 2.0 / 시네마틱 → Cinema Studio 3.0·Veo 3.1
- 15초 초과면 체이닝(`references/master-prompt.md`) 여부

## 출력 형식 (항상 이 5개 섹션)

### 1. 장면 분석
- 표: | # | 시간 | 피사체·동작 | 배경 | 카메라 | 조명·색감 | 스타일 |
- **1-1 자막/텍스트**: 화면에 박힌 자막을 텍스트화. 단, 분석기는 별도 OCR을 주지 않으므로 확인된 화면 텍스트(로고·박힌 문구)와 **음성(VO) 대사 전문**을 함께 제공하고, 어느 쪽인지 표시한다.
- **1-2 목소리 분석**: 분석기는 음향 측정값을 주지 않는다. 내용 기반 **추정 프로필**로 제공 — 성별, 추정 연령대, 언어, 톤/무드(예: 차분·세련), 말 속도, 발성 특징. "추정"임을 명시한다. (재현 시 VO 매칭·voice 생성에 사용)

### 2. 제품 스왑 호환성 리뷰
- 원본 제품 형태 판정 → 사용자 제품과 형태가 맞는지 리뷰(가능/불가/조건부). 불가면 이유와 대안(형태 맞는 레퍼런스) 제시.

### 3. 첨부 이미지 및 보완자료 가이드
- 위 "제품 스왑 & 첨부 자료 가이드"의 폼팩터별 체크리스트를 이 영상에 맞게 구체화해 요청 목록으로 제시.

### 4. 재현용 마스터 프롬프트 (상세)
generate_video / ad-video-helper에 바로 투입할 완결형. **반드시 아래를 상세히 포함**:
- 인물 디테일: 성별·연령·피부·헤어(색/스타일)·의상·액세서리·표정·손/네일.
- 배경/공간: 장소·소품·벽/바닥·분위기.
- 조명·컬러그레이드: 광원 방향·키/필·톤(웜/쿨)·채도·필름룩.
- 카메라: 샷 사이즈·앵글·무빙(핸드헬드/정적/트래킹)·심도, 구간별 동선.
- 목소리/오디오: 추정 VO 프로필대로 성별·톤·속도·언어 지시 + BGM 성격.
- 페이싱: 장면별 초 단위 타이밍.
- 제품 스왑 규칙: `{첨부제품}`으로 교체, 형태·라벨·색상 100% 유지, 사용법 매칭.
- ≤15초 원테이크 / >15초 체이닝(master-prompt.md).

마지막에 "이 프롬프트로 만들까요? (생성은 크레딧이 듭니다)"로 모드 A 연결 제안, 승인 전 생성 금지.

마지막에 "이 프롬프트로 바로 만들어 드릴까요? (생성은 크레딧이 듭니다)"로 모드 A 연결을 제안하되, 승인 전에는 생성하지 않는다.

## 제품 스왑(swap) & 첨부 자료 가이드

재현 프롬프트는 기본적으로 **원본 제품을 사용자의 실제 제품으로 교체**할 수 있어야 한다. 프롬프트에 `{첨부제품}` 변수와 "형태·라벨·색상을 첨부 이미지와 100% 동일 유지" 규칙을 넣는다.

**대전제 — 폼팩터(형태)가 맞아야 스왑이 자연스럽다.** 원본 제품의 형태와 사용법을 먼저 파악하고, 사용자 제품이 같은 형태인지 확인한다. 다르면 경고한다.

| 원본 형태 | 첨부해야 할 제품 | 형태 불일치 시 |
|------|------|------|
| 스틱/립스틱 | 스틱·립·멀티스틱 | 손동작 붕괴 → 형태 맞는 다른 레퍼런스 권유 |
| 단지형 크림 | 단지형 크림 | 떠바르는 동작 안 맞음 |
| 튜브형 | 튜브형 | 짜는 동작 안 맞음 |
| 펌프/보틀 | 펌프/보틀 | 펌프 동작 안 맞음 |
| 팩트/쿠션 | 팩트/쿠션 | 여닫기·퍼프 동작 안 맞음 |

**첨부 이미지 요구(최소 1 / 권장 3~5장):**
- 공통 필수: 정면 라벨 컷 — 고해상·선명, 균일 조명, 단순 배경, 라벨 글자 판독, 필터 최소.
- 립스틱/스틱: 닫힘 정면 + 뚜껑 열어 벌브(색) 노출 + 45°.
- 단지형 크림: 정면 라벨 + 뚜껑 열어 제형 + 45°.
- 튜브형: 정면 라벨 전체 + 캡/노즐 클로즈업 + 짜낸 제형.
- 펌프/보틀: 정면 + 펌프 헤드 + 측면.
- 원본에 손에 든/사용 장면이 있으면 손에 든 스케일 컷 1장 추가.

**절차:** 분석으로 원본 제품 형태를 판정 → 사용자에게 "이 영상은 [형태]형입니다. 같은 형태의 제품 사진 [체크리스트]를 올려주세요"라고 첨부 목록을 명시적으로 요청 → `media_upload_widget`으로 받는다 → 스왑 프롬프트의 `{첨부제품}`에 연결. 형태가 안 맞으면 먼저 알리고 형태 맞는 레퍼런스를 권한다.
